What is the Google Indexing API? A Deep Dive

1. Definition: How Does It Work?

The Google Indexing API is a RESTful API introduced by Google in 2021 as part of its Search Console API suite. It allows website owners to submit URLs for indexing directly to Google, bypassing the traditional URL Inspection Tool and Change of Address workflows.

Unlike the older Search Console API, which required manual submissions via the dashboard, the Indexing API automates the process, making it ideal for:

  • E-commerce sites with frequent product updates
  • Blogs and news sites publishing daily content
  • Dynamic websites with user-generated content
  • Large-scale migrations (e.g., domain changes, site restructures)

2. Key Features of the Google Indexing API

Feature Description
Bulk URL Submission Submit thousands of URLs at once (up to 10,000 per request).
Asynchronous Processing Google processes requests in the background, reducing latency.
Real-Time Indexing Faster than manual submissions via Search Console.
Error Handling & Retries Built-in mechanisms to retry failed submissions.
Integration with Google Cloud Works seamlessly with Google Cloud Functions, BigQuery, and other tools.
No Rate Limits (Soft Limits) While Google doesn’t enforce strict limits, abuse can lead to temporary bans.

3. How It Differs from Traditional Indexing Methods

Method Speed Automation Bulk Submission Best For
Google Search Console (Manual) Slow (hours/days) Manual No Small sites, one-off updates
Sitemaps (XML/HTML) Moderate (depends on crawl budget) Semi-automated Yes Static content, blogs

4. Who Should Use the Google Indexing API?

The API is not a replacement for sitemaps but rather a complementary tool for: 🔹 E-commerce platforms (Shopify, WooCommerce, Magento) with daily product updates 🔹 News publishers (WordPress, Medium) that need real-time indexing 🔹 Dynamic content sites (forums, SaaS dashboards, membership portals) 🔹 Websites undergoing major migrations (domain changes, URL restructures) 🔹 SEO agencies managing multiple client sites

Step-by-Step Guide: How to Set Up the Google Indexing API

Before diving into implementation, you’ll need two essential components:

  1. A Google Cloud Project (for API access)
  2. A backend system (Node.js, Python, PHP, etc.) to send requests

Let’s walk through the full setup process.

Step 1: Create a Google Cloud Project & Enable the API

1.1 Sign Up for Google Cloud (If You Don’t Have an Account)

  • Go to Google Cloud Console.
  • Click "Select a project" → "New Project".
  • Enter a project name (e.g., "UltraStaticSEO-IndexingAPI").
  • Click "Create".

1.2 Enable the Indexing API

  • In the Google Cloud Console, navigate to "APIs & Services" → "Library".
  • Search for "URL Inspection API" (this is the official name of the Indexing API).
  • Click "Enable".

1.3 Create API Credentials

  • Go to "APIs & Services" → "Credentials".
  • Click "Create Credentials" → "OAuth client ID" (for web applications).
  • Fill in:
    • Application type: Web application
    • Name: UltraStaticSEO-IndexingAPI
    • Authorized JavaScript origins: https://yourdomain.com (or http://localhost for testing)
    • Authorized redirect URIs: https://yourdomain.com/auth/callback (or http://localhost/auth/callback)
  • Click "Create".
  • Copy the Client ID and Client Secret—you’ll need them later.

1.4 Generate an Access Token

The Indexing API requires an OAuth 2.0 access token. Here’s how to generate one:

  1. Use Google’s OAuth Playground (https://developers.google.com/oauthplayground):
    • Select "Use your own OAuth credentials".
    • Paste your Client ID and Client Secret.
    • Set the Scopes to:
      https://www.googleapis.com/auth/www-authenticate
https://www.googleapis.com/auth/url-inspection
    • Click "Authorize APIs", then "Exchange token".
    • Copy the Access Token—this will be used in API requests.

Step 2: Send Your First Indexing Request

Now that you have API access, let’s structure a sample HTTP request to submit a URL for indexing.

2.1 Required Headers & Body

Every request must include:

  • Authorization Header (Bearer YOUR_ACCESS_TOKEN)
  • Content-Type: application/json
  • Request Body (JSON format):
{
  "url": "https://yourwebsite.com/new-page",
  "type": "URL"
}

2.2 Example cURL Request

curl -X POST \
  https://urlinspection.googleapis.com/v1/urlNotifications:notify \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://yourwebsite.com/new-page",
    "type": "URL"
  }'

2.3 Expected Response

A successful request returns:

{
  "url": "https://yourwebsite.com/new-page",
  "status": "SUCCESS",
  "notificationId": "1234567890abcdef1234567890abcdef"
}

2.4 Handling Errors

If Google rejects your request, you’ll get a response like:

{
  "error": {
    "code": 400,
    "message": "Invalid URL format",
    "details": [
      {
        "type": "type.googleapis.com/google.rpc.BadRequest",
        "detail": "URL must be a valid HTTPS URL"
      }
    ]
  }
}

Common error codes & fixes:

Error Code Meaning Solution
400 Invalid URL format Ensure URL starts with https://
401 Unauthorized Regenerate access token
403 Quota exceeded Wait or increase quota
429 Too many requests Implement retry logic

Step 3: Automate Indexing with a Backend Script

For real-world use, you’ll need a backend system to:

  • Generate access tokens (refreshing them every 1 hour)
  • Batch submit URLs (for efficiency)
  • Handle retries (if Google fails to process a request)

3.1 Example in Node.js (Using Axios)

const axios = require('axios');
const { OAuth2Client } = require('google-auth-library');

const CLIENT_ID = 'YOUR_CLIENT_ID';
const CLIENT_SECRET = 'YOUR_CLIENT_SECRET';
const REDIRECT_URI = 'https://yourdomain.com/auth/callback';

const oauth2Client = new OAuth2Client(CLIENT_ID, CLIENT_SECRET, REDIRECT_URI);

// Generate a new access token
async function getAccessToken() {
  const token = await oauth2Client.getAccessToken();
  return token.token;
}

// Submit a URL for indexing
async function submitUrlForIndexing(url) {
  const accessToken = await getAccessToken();

  const response = await axios.post(
    'https://urlinspection.googleapis.com/v1/urlNotifications:notify',
    {
      url: url,
      type: 'URL'
    },
    {
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/json'
      }
    }
  );

  return response.data;
}

// Example usage
submitUrlForIndexing('https://yourwebsite.com/blog/new-post')
  .then(data => console.log('Success:', data))
  .catch(err => console.error('Error:', err.response.data));

Step 4: Integrate with Your CMS or E-Commerce Platform

Now that you have a working script, let’s explore how to integrate it with popular platforms.

4.1 WordPress (Using a Plugin or Custom Code)

If you’re using WordPress, you can:

  • Use a plugin like "Google Indexing API for WordPress" (if available).
  • Hook into save_post and submit new posts automatically.

Example (PHP in functions.php):

add_action('save_post', 'submit_post_to_indexing_api', 10, 2);

function submit_post_to_indexing_api($post_id, $post) {
    if (wp_is_post_revision($post_id) || wp_is_post_autosave($post_id)) {
        return;
    }

    $url = get_permalink($post_id);
    $access_token = get_google_access_token(); // Your OAuth function

    $args = [
        'headers' => [
            'Authorization' => 'Bearer ' . $access_token,
            'Content-Type' => 'application/json'
        ],
        'body' => json_encode([
            'url' => $url,
            'type' => 'URL'
        ])
    ];

    $response = wp_remote_post(
        'https://urlinspection.googleapis.com/v1/urlNotifications:notify',
        $args
    );

    if (is_wp_error($response)) {
        error_log('Google Indexing API Error: ' . $response->get_error_message());
    }
}

4.2 Shopify (Using Liquid & API Webhooks)

For Shopify stores, you can:

  • Trigger indexing when a product is updated via Shopify Admin API.
  • Use a custom app to send requests to the Indexing API.

Example (JavaScript in a Shopify App):

// When a product is updated via Shopify Admin
async function handleProductUpdate(product) {
  const accessToken = await getGoogleAccessToken();
  const url = `https://yourshopify.com/products/${product.id}`;

  const response = await fetch(
    'https://urlinspection.googleapis.com/v1/urlNotifications:notify',
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ url, type: 'URL' })
    }
  );

  if (!response.ok) {
    console.error('Indexing failed:', await response.text());
  }
}

4.3 Custom Web Applications (Next.js, React, etc.)

If you’re building a custom SaaS or dynamic site, you can:

  • Trigger indexing on content changes (e.g., after a user submits a form).
  • Use a serverless function (AWS Lambda, Google Cloud Functions) to send requests.

Example (Next.js API Route):

// pages/api/submit-indexing.js
export default async function handler(req, res) {
  const { url } = req.body;

  if (!url) {
    return res.status(400).json({ error: 'URL is required' });
  }

  try {
    const accessToken = await getGoogleAccessToken();
    const response = await fetch(
      'https://urlinspection.googleapis.com/v1/urlNotifications:notify',
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${accessToken}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({ url, type: 'URL' })
      }
    );

    const data = await response.json();
    res.status(200).json(data);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
}
